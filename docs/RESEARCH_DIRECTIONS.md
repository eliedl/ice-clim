## Concern of this document

Speculative research directions surfaced during literature reading that are **not
methodology decisions for the current climatology pipeline**, and therefore do
not belong in [LITERATURE.md](LITERATURE.md) (whose concern is pipeline-feeding
methodology) or [DECISIONS.md](DECISIONS.md).

This is where forward bets are captured so they are not lost — kept walled off
from the methodology synthesis. Entries reference [READING_LOG.md](READING_LOG.md)
by `eNNN` pointer; the log stays the permanent atomic store (nothing is moved).

Promote a thread out of this file into LITERATURE.md + DECISIONS.md only once it
becomes an actual methodological choice for the pipeline.

### Timestamping convention

Same as LITERATURE.md: each direction carries a date; re-aggregation appends a
new dated `### YYYY-MM-DD` block rather than overwriting.
